# invalid-encoding-test

Test fixtures for REACHABLE's SAST parser resilience.

## Purpose

These files exercise the scanner's behaviour when it encounters files it
cannot parse.  The expected behaviour is:

1. **No crash** — the scan completes normally
2. **Skip + report** — the file appears in `08-coverage.json` under
   `files.sast_skipped_files` with a clear reason
3. **Coverage gap** — `files.sast_unparseable` count > 0

None of these files should produce secret/CWE/malware findings because
semgrep will reject them before pattern matching runs.

## Setup — run once after clone

Fixtures 2 and 3 contain non-UTF-8 bytes that cannot be stored in a
text-based VCS without corruption.  Generate them with:

```bash
cd /path/to/reach-testbed
python3 invalid-encoding-test/create_test_fixtures.py
```

This creates (or overwrites):
- `latin1_no_declaration.py`   — Latin-1 encoded, no coding declaration
- `binary_disguised_as_python.py` — ELF magic + null bytes

## Test cases

| File | Encoding issue | Why semgrep rejects it |
|------|---------------|------------------------|
| `bytes_literal_with_em_dash.py` | Em dash (U+2014) inside `b"""..."""` | Python bytes literals are ASCII-only (PEP 3120). Committed to VCS as UTF-8 — em dash is in the file but inside a `b""` literal which Python rejects. |
| `latin1_no_declaration.py` | Latin-1 bytes, no `# -*- coding:` header | Python 3 defaults to UTF-8. Latin-1 bytes 0x80–0xFF are not valid UTF-8. **Must be generated by `create_test_fixtures.py`.** |
| `binary_disguised_as_python.py` | ELF header + null bytes in a `.py` file | Null bytes cause an immediate `SyntaxError`. Simulates a corrupted or misnamed binary. **Must be generated by `create_test_fixtures.py`.** |

## Expected scan results (ground truth)

`reach-testbed/expected-results/invalid-encoding-test.json`:
```json
{
  "sast_unparseable": 3,
  "sast_skipped_files": 3
}
```

## Data flow

```
semgrep errors[] → _extract_skipped_from_errors()
                → scanner._skipped_files
                → get_skipped_files()
                → _merge_sast_skipped(raw_dir, skipped)   # _step_secrets + _step_malware
                → raw/sast-skipped.json
                → pipeline_finalize loads it
                → run_coverage_analysis(sast_skipped_files=...)
                → 08-coverage.json  files.sast_unparseable / files.sast_skipped_files
```

## Roadmap

**Sanitize-and-reparse (advanced mode):**
1. Shadow-copy each skipped file to a temp path
2. Detect encoding with `chardet` / `charset-normalizer`
3. Re-encode to UTF-8 with `errors='replace'`
4. Re-run semgrep on sanitized copy
5. Promote findings back with `"sanitized_parse": true` flag

This would recover findings from files like Latin-1-encoded legacy PHP,
Windows-1252 config files, and setup.py with non-ASCII docstrings.
